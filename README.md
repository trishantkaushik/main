This project is a minimal personal portfolio website developed using HTML with an external CSS file for styling. The structure of the webpage begins with the <html> tag, which acts as the root element. The <head> section contains the page title and links to external resources, including the Google Fonts library for clean typography and a separate style.css file for design and layout. The <title> tag sets the browser tab name as “My Portfolio”.

The <body> section contains the visible content of the website. A <header> element is used to introduce Trishant Kaushik, displaying his name as the main heading along with a short description as a BCA student interested in web development. Below the header, a simple <nav> section provides internal navigation links such as Home, About, Projects, and Contact. These links allow smooth navigation between different sections of the same webpage using section IDs.

The webpage is divided into semantic <section> elements to maintain clarity and structure. The Home section provides a short introduction and may include basic interests or hobbies. The About section briefly describes academic background and learning goals. The Projects section is designed to showcase personal or academic work. Each section is organized clearly to maintain a clean and minimal layout.

assignment 4
This project implements an interactive Frequently Asked Questions (FAQ) section using HTML, CSS, and JavaScript with an object-oriented approach. HTML provides the basic page structure and a container where all questions and answers are dynamically inserted. CSS is used to design the layout and hide answers by default while adding visual cues like “+” and “−” icons for expansion and collapse. JavaScript defines a QAItem class that stores each question and answer and contains a method to render them into HTML elements. Event listeners are attached to each question so that when the user clicks it, the corresponding answer toggles its visibility. The questions and answers are stored in an array of QAItem objects, allowing the FAQ section to be dynamically generated and easily extended.
